Local-first Codex workspace

把散落的 Codex 扩展，变成可管理资产。

Codex Extension Hub 一键扫描本机 Plugin、Skill、MCP 和 App，用本地网页告诉你装了什么、怎么调用、哪里重复、哪个适合当前任务。

查看 GitHub 阅读教程

安装不应该成为门槛。

教程面向第一次接触本项目的用户：复制一行命令，安装依赖，扫描本机扩展，然后打开本地网页。

One-command install 
curl -fsSL https://raw.githubusercontent.com/Daviddwt/codex-extension-hub/main/scripts/install.sh | bash

cd ~/codex-extension-hub
npm run dev

推广时讲清楚四件事。

这不是“又一个插件列表”，而是把本机扩展环境变成可以搜索、筛选、推荐和维护的工作台。

看清现状

按 Plugin、Skill、MCP、项目级、全局和异常状态快速理解本机扩展结构。

按任务推荐

输入要做的事，本地推荐器根据名称、用途、分类和提示词给出候选扩展。

发现异常

同名扩展、元数据缺失、marketplace 不一致都会被标记出来，方便清理。

检查更新

第三方 marketplace 插件可检查更新，自建和系统托管插件按策略跳过。

真实边界

只读扫描元数据，不执行插件脚本，不启动 MCP，不上传本地数据。

团队可复用

适合沉淀团队常用 skill/plugin 命名、用途说明和调用入口。

用户路径要短。

文章和 README 都围绕这条路径组织：装上、打开、搜索、筛异常、做更新。

01 Install

一键安装脚本克隆仓库、安装依赖并生成本机扫描结果。

02 Open

本地打开 127.0.0.1，只在自己的电脑上查看扩展数据。

03 Search

按名称、用途、调用名、项目路径或 marketplace 搜索。

04 Maintain

检查异常项和第三方更新，把扩展环境整理干净。
